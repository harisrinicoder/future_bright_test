Django Templates let you create dynamic HTML pages by combining HTML with Django Template Language (DTL).

üìù What is a Django Template?

A template is an HTML file that contains placeholders for dynamic data. Django renders these templates by replacing placeholders with actual values passed from views.

üîß Key Features of Django Templates

  • Template Variables: Insert dynamic data using double curly braces: {{ variable_name }}
  • Template Tags: Logic and control flow using tags like {% if %}, {% for %}, and {% block %}
  • Template Inheritance: Create base templates and extend them to reuse common layout parts.
  • Filters: Modify variables output, e.g., {{ name|upper }} makes text uppercase.

üìÇ Example Template (home.html)

<!DOCTYPE html>
<html>
  <head>
    <title>Welcome to {{ site_name }}</title>
  </head>
  <body>
    <h1>Hello, {{ user.username }}!</h1>

    {% raw %}
    {% if posts %}
      <ul>
        {% for post in posts %}
          <li>{{ post.title }} - {{ post.published_date }}</li>
        {% endfor %}
      </ul>
    {% else %}
      <p>No posts found.</p>
    {% endif %}
    {% endraw %}

  </body>
</html>

üîó How to Use Templates in Views

Use Django's render() shortcut to load templates and pass context data: 

from django.shortcuts import render

def home_view(request):
    context = {
        'site_name': 'Future Bright Institute',
        'user': request.user,
        'posts': Post.objects.all(),
    }
    return render(request, 'home.html', context)

üìö Summary

  • Templates separate your Python logic from HTML presentation
  • They use DTL syntax with variables, tags, and filters
  • Template inheritance helps keep your code DRY (Don't Repeat Yourself)
  • The render() function sends data from views to templates
