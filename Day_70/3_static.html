🗂️ Django Static Files – Common & App-Specific

Static files are CSS, JavaScript, images, or fonts used by your Django website. You can have global static files or app-specific ones.

⚙️ Step 1: Configure settings.py

# settings.py

# URL to use when referring to static files
STATIC_URL = "/static/"

# Global static folder
STATICFILES_DIRS = [BASE_DIR / "static"]

# For collecting all static files
STATIC_ROOT = BASE_DIR / "staticfiles"

📁 Step 2: Folder Structure

project_root/
├── static/                # Global static files
│   ├── css/
│   │   └── style.css
│   ├── js/
│   │   └── scripts.js
│   └── images/
│       └── logo.png

├── blog/
│   └── static/
│       └── blog/
│           └── blog.css

├── courses/
│   └── static/
│       └── courses/
│           └── courses.js

📝 Step 3: Load Static in Templates

Always add {% load static %} at the top of your template.

📄 Example from base.html

{% raw %}
{% load static %}



{% endraw %}

📄 Example from blog/templates/blog/home.html

{% raw %}
{% load static %}

{% endraw %}

🧪 Step 4: Run Dev Server and View

python manage.py runserver

Static files are automatically served in development mode. Visit your pages to confirm CSS and JS are applied.

📦 Step 5: Collect Static Files for Deployment

python manage.py collectstatic

This command collects all static files into STATIC_ROOT (usually staticfiles/) for production use.

🔁 Summary

  • STATICFILES_DIRS — for global static assets like main CSS/JS
  • app/static/appname/ — for app-specific static assets
  • Use {% static 'path/to/file' %} to load files in HTML
  • Use collectstatic before deploying your site
Always organize static files clearly to make your project scalable and maintainable.
